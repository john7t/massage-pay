# 薪資追蹤系統

部署於 GitHub Pages 的純前端薪資追蹤 Web App，供按摩師記錄每日工作量與薪資計算。資料存於本機瀏覽器（localStorage），離線優先，無需後端伺服器。

> **目前版本：v1.9-012**（版號內嵌於各頁 `<title>` 與 `common.js` 檔頭註解）。

## 專案簡介

系統以「純前端 + 本機儲存」為核心設計，讓門市老師（按摩師）能離線記錄每日支數、薪資、客管資料。多人協作的部分（權限審核、員工名單、統計發布）透過 GitHub 作為輕量雲端層交換資料，但老師端的日常記帳完全不依賴網路。

主要使用者為越南籍在台留學生，因此全系統支援中文／越南文雙語切換。門市為 24 小時跨夜營業，系統內建營業日切點邏輯處理跨夜班別。

## 架構總覽（多頁面）

系統由數個獨立頁面 + 一個共用邏輯檔組成，彼此透過 localStorage、URL hash 與雲端設定檔交換資料：

| 檔案 | 用途 | 對應使用者 |
|------|------|------------|
| `index.html` | 主程式：首頁、月份明細、年度薪資、設定頁殼層 | 一般老師 |
| `settings.js` | 設定頁各子分頁元件（由 `index.html` 引入，降低單檔體積） | — |
| `auth.html` | 權限後台：老師/主管啟用、主管審核、管理者控台、員工名單、上工時段圖表 | 主管／管理者 |
| `booking.html` | 自約行事曆：老師自行登記客人預約、衝突偵測 | 一般老師 |
| `common.js` | 共用工具庫（i18n、薪資與流水計算、資料存取等），以 `window.MP` 供各頁共用 | — |

## 功能總覽

### 首頁
- 今日狀態燈號（未開班／營業中／請假以不同顏色燈號表示）與快捷新增支數入口
- 前一筆紀錄快速參考，點擊今日或前一筆可開啟當日編輯面板
- **營業日切點**：跨夜班別的凌晨紀錄自動歸入前一個營業日，避免大夜班跨日算錯
- 底部「設定」分頁在基本資料／店家資訊有未確認變更、或工作證即將到期（30 天內）時顯示紅色 ❗ 提醒

### 月份明細
- 每日一列：日期、星期、支數、薪水、老點、狀態
- 上半月（1–15 日）／下半月（16 日–月底）分期彙總
- 點擊展開每日編輯面板
- **狀態標記**：正常／病假／遲到／休假／事假／早退
- **老點**：回頭客標記（由流水逐筆累計）
- **技能加計**：刮痧、拔罐、修腳皮，可於設定頁啟用

### 流水記錄（客管明細）
每日編輯面板的「流水」清單，每筆流水可記錄：
- 服務項目（F1/B1/F2/B2/FB2/F3/B3/FB3）與加鐘數
- 支數、老點數
- 開始時間（可手動修正）
- 客人姓名、稱謂、電話
- 受力強度（輕／普通／重，分身體與腳兩組）、加強部位（頭/肩/上背/下背/腰/臀/大腿/小腿/手臂）
- 客人要求（指定男/女、台籍/越籍/中籍）
- 本次一起出牌老師（老點出牌／輪班出牌編號）
- 自訂標籤（hashtag）

流水中若填了客人姓名／電話，會自動寫入**本機客戶資料庫**，下次同一客人可搜尋帶入慣用的受力與部位設定。

### 年度薪資
12 個月彙總表（支數、薪水、老點、小計）與全年合計。

### 設定頁（子分頁式）
| 子分頁 | 內容 |
|--------|------|
| 首頁設定 | 老師代碼、每支單價、語系、年份切換、主題色 |
| 基本資料 | 個人資料、就讀學校、工作許可證資訊；以「申請碼 → 確認碼」流程送出 |
| 店家資訊 | 所屬門市、班別、上下班時間、服務選項等；同樣走申請碼／確認碼流程 |
| 預約 | 導向 `booking.html` |
| 客戶管理 | 搜尋歷史客戶、近期客戶清單、編輯客戶固定屬性、標籤統計 |
| 公告 | 公司公告中心（開發中） |
| 圖表統計 | 讀取統計快照，顯示班別比例、性別比例、上工人數時段分布 |
| 建議箱 | 意見回饋（開發中） |
| 備份 | 匯出／匯入備份碼 + 完整 JSON 備份 |
| 違規紀錄 | 個人違規紀錄（預留功能） |
| 管理 | 導向 `auth.html` 權限後台 |

- **主題系統**：黑／粉紅／白／灰四種主題，淺色主題另有專屬文字對比配色
- 三則重要提示（資料本機保存、LINE 與瀏覽器暫存分開、換機前需匯出）

## 權限管理系統

採三層架構：**老師 → 主管 → 管理者**。主管／管理者操作介面集中於獨立的 `auth.html`，老師端只保留「管理」入口連結。

### 老師
- 輸入老師編號 → 產生驗證字串 → 管理者提供啟動碼 → 完成啟用
- 可申請主管權限
- 可透過「申請碼 → 確認碼」流程送出基本資料／店家資訊變更

### 主管
- 經審核後取得主管權限
- 可產生老師啟動碼、審核其他老師的主管申請
- 可查看主管名單、操作記錄（限老師與主管層級）
- **員工名單**：檢視所屬老師名單，編輯其班別、上下班時間、所屬門市
- **申請碼處理**：處理老師送出的基本資料／店家資訊申請碼，產生確認碼交還老師
- **上工時段圖表**：呈現所屬門市老師的上工分佈

### 管理者
- 新增／刪除管理者、審核主管申請、產生啟動碼、取消主管權限
- 查看完整操作記錄
- 管理全店員工名單、發布統計快照
- GitHub 連線設定

## 自約行事曆（`booking.html`）

老師個人備忘用的預約登記（非門市正式訂位系統）：
- 行事曆／事件兩種檢視
- 新增自約：服務項目、加鐘、指定客人、指定服務老師（不指定／按代碼／按條件）
- **衝突偵測**：時間重疊時提示，仍可選擇強制建立
- **請假日偵測**：當天已標記病假／休假／事假時提示
- 預約狀態機與完整異動時間軸

## 圖表統計

- 主管／管理者可發布統計快照供全店老師讀取
- 老師端顯示：班別比例、性別比例、上工人數／在線人數時段分布
- 非主管／管理者身份時圖表僅供檢視

## 完整備份

- **完整 JSON 備份**：匯出設定、主題、全年資料（含流水）、自約為單一 `.json` 檔，內含版本標記與校驗碼防止竄改後誤匯入
- **JSON 匯入**：驗證來源與校驗碼，且僅允許匯入相同老師代碼的備份
- 另有輕量「範圍備份碼」（壓縮文字碼，僅含月份薪資數字）

## 錯誤兜底機制

各頁在最早期插入全域錯誤攔截器：
- 攔截未捕捉的例外與 React 渲染例外
- 觸發時顯示雙語（中／越）「網站更新中」全螢幕提示與錯誤代碼，方便回報
- 提示文字強調「本機資料仍保留」，避免使用者誤以為資料遺失

## 技術架構

- **前端**：React 18 + Babel（瀏覽器端編譯）+ Tailwind CSS CDN
- **儲存**：localStorage（按老師代碼與月份分 key）
- **部署**：GitHub Pages 靜態託管
- **雲端層**：透過 GitHub API 讀寫設定檔，作為多人協作的輕量後端
- **主題**：CSS `data-theme` 屬性 + 對應純 CSS 背景色定義，避免部分瀏覽器 JS 動態改色不即時重繪的問題
- **雙語**：中文／越南文，i18n 字典集中於 `common.js`

## 部署方式

1. 建立 GitHub repo
2. 上傳所有頁面檔案與必要的設定檔
3. Settings → Pages → Source 選 `main` → Save
4. 等待 1–2 分鐘，即可透過 `https://<帳號>.github.io/<repo名>/` 存取

### 部署注意事項

- **所有頁面共用 `common.js`**：改動 `common.js` 時，所有頁面都受影響，務必一起部署並清除瀏覽器快取，否則會出現新舊版本混用的錯誤。
- **瀏覽器快取**：更新後若行為異常，多半是 `common.js` 被快取，需強制重新整理。
- **LINE 與瀏覽器暫存分開**：提醒使用者固定使用同一個瀏覽器，否則啟動碼與資料不互通。

## 版號規則

- 所有頁面共用一個版號（格式 `v1.9-XXX`）。
- 平常版更：只更新有改動的檔案版號。
- 版號尾數逢 0（如 020、030）：所有檔案版號全部同步對齊一次，作為乾淨的還原基準點。
