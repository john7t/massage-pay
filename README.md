# 薪資追蹤系統

部署於 GitHub Pages 的純前端薪資追蹤 Web App，供按摩師記錄每日工作量與薪資計算。資料存於本機瀏覽器（localStorage），無需後端伺服器。

## 功能總覽

### 首頁

- 顯示今日狀態與快捷新增入口
- 昨日/上一筆紀錄快速參考

### 月份明細

- 每日一列：日期、星期、支數、薪水、老點、狀態
- 上半月（1–15 日）/ 下半月（16 日–月底）分期彙總
- 點擊展開每日編輯面板
- **支數雙模式**：直接輸入總數 或 分組輸入逐組客人數再加總
- **狀態標記**：正常 / 病假 / 遲到 / 休假 / 事假 / 早退
- **老點**：當日回頭客組數
- **技能加計**：刮痧（$250）、拔罐（$250）、修腳皮（$300），可於設定頁啟用
- 一鍵清除：重置當日所有資料

### 年度薪資

- 12 個月彙總表：支數、薪水、老點、小計
- 底部全年合計

### 設定頁

- 老師代碼、每支單價、語系（中文 / 越南文）、年份切換
- 技能項目啟用開關
- 裝置 ID 顯示
- 備份功能：匯出 / 匯入備份碼（含技能資料）
- 三則重要提示（資料本機保存、LINE 與瀏覽器暫存分開、換機前需匯出）

## 權限管理系統

採三層架構：**老師 → 主管 → 管理者**

### 老師（一般使用者）

- 輸入老師編號 → 產生加密字串 → 管理者提供啟動碼 → 完成啟用
- 可於設定頁申請主管權限

### 主管

- 由管理者或其他主管審核申請碼後發放主管啟動碼
- 可產生老師啟動碼
- 可審核其他老師的主管申請
- 可查看主管名單並取消他人主管權限
- 可查看操作記錄（僅限老師與主管層級的記錄）

### 管理者

- 透過 `admin.cfg` 檔案中的雜湊碼驗證身份
- 新增 / 刪除管理者（透過 GitHub API 自動寫入）
- 審核主管申請、產生啟動碼
- 取消主管權限（自動寫入 `revoked.cfg`）
- 查看完整操作記錄（含操作人編號）
- 加密 / 解密工具
- GitHub 設定（帳號、Repo、Token），儲存後自動鎖定，需按「編輯」才可修改

### 撤銷機制

- 管理者或主管點擊「取消主管權限」→ 自動寫入 `revoked.cfg`（需設定 GitHub Token）
- 被撤銷者下次開啟主管頁面時自動偵測並降級，顯示「權限已過期」

## 操作記錄

記錄分為三個層級，依角色分層可見：

| 層級 | 可見範圍 | 記錄內容 |
|------|----------|----------|
| 老師 | 老師、主管、管理者 | 啟用主管權限 |
| 主管 | 主管、管理者 | 產生啟動碼、審核主管申請、取消主管權限 |
| 管理者 | 僅管理者 | 新增/刪除管理者、管理級操作（含操作人編號） |

## 檔案結構

```
massage-pay/
├── index.html      ← 主程式（單一 HTML，含 React + Tailwind）
├── admin.cfg       ← 管理者清單（第一行為隨機字串）
├── revoked.cfg     ← 已撤銷主管的雜湊碼（第一行為隨機字串）
├── key.cfg         ← 密鑰種子（隨機字串）
└── README.md
```

### admin.cfg 格式

```
加密編號:雜湊碼
```

例：`xxxxxx:yyyyyy` → 管理頁面自動解密前段顯示老師編號，後段為不可逆雜湊碼

舊格式（僅雜湊碼）仍可用於身份驗證，但管理頁面顯示為 `—`。

### revoked.cfg 格式

```
# Revoked supervisor hashes (one per line)
雜湊碼
```

## 技術架構

- **前端**：React 18 + Babel（瀏覽器端編譯）+ Tailwind CSS CDN
- **儲存**：localStorage（按老師代碼與月份分 key）
- **部署**：GitHub Pages（靜態託管）
- **GitHub API**：透過 Personal Access Token 讀寫 `admin.cfg` / `revoked.cfg`
- **加密**：XOR 可逆加密（動態密鑰，從多個 cfg 檔案的隨機字串中提取拼接）+ FNV 雜湊（管理者驗證、撤銷驗證）

## 密鑰機制

XOR 加密的密鑰不寫死在原始碼中，而是由管理者設定從多個檔案中提取片段拼接而成：

- 每個 cfg 檔案的第一行含有 20–60 字元的隨機字串
- 管理頁面的「密鑰設定」可設定 3–4 條提取規則
- 每條規則指定：檔案名稱、起始位置、取幾字
- 各片段依序拼接成 XOR 密鑰
- 未設定時使用內建預設值

## 部署方式

1. Fork 或建立 GitHub repo
2. 上傳 `index.html`、`admin.cfg`、`revoked.cfg`
3. Settings → Pages → Source 選 `main` → Save
4. 等待 1–2 分鐘，即可透過 `https://<帳號>.github.io/<repo名>/` 存取

## GitHub Token 設定

管理頁面的 GitHub 自動寫入功能需要 Personal Access Token：

1. 前往 `https://github.com/settings/tokens?type=beta`
2. Generate new token → 選擇對應 repo
3. Repository permissions → Contents：Read and write
4. 產生後貼到管理頁面的 GitHub 設定
